## The 13 additions

### Group A — Anti-patterns (3)

**A18: Recovery handlers narrowed to wrong exception class.** `except RuntimeError` when production hits `TimeoutError`. Recovery path silently never fires. Fix: catch broadest applicable `Exception` + `isinstance` at re-raise to preserve type-specific error context.

**A19: Parallel template/code branches drift over multiple sessions.** Today vs past, new vs archived, primary vs secondary loops edited in isolation diverge. Fix: refactor to shared macro/component (structural parity) OR audit for parity periodically. Drift is invisible until specific input shape exercises the missed path.

**A20: Redesigning presentation layer while data layer is unstable.** Beautiful UI rendering "Source unreachable" half the time. Gate redesigns behind data-layer stability. Test: would redesigned UI render correctly across empty-state, partial-data, and degraded-source cases the current pipeline produces?

All 3 added to consulting playbook anti-patterns library + code-correctness audit prompt hunt categories.

### Group B — Consulting playbook section updates

**§6 Tier-Based Shipping — add: Phase-numbered plans become commit-message scaffolding.** When implementation plans number phases (Phase 0, 1, ...), commits adopt the numbering and `git log --oneline` becomes phase-progression view. Recommend numbered phases in all SignalWorks implementation plans.

**§7 Multi-Agent Audit Discipline — add: Trust-but-verify subagent factual claims.** Before acting on any agent claim about production state ("X is broken", "Y is missing"), run `grep`/`curl`/`jq` one-liner verification. Cost is seconds; rework on wrong claim is hours. Especially important with role-specialized agents working from a slice of context.

**§13 Pre-Delivery Sweep — add: Iterative ship-then-sweep cycles between "verifier passed" and "delivery sent".** Pre-launch evaluation gates structural correctness; post-deploy sweeps catch surface-level production bugs that consistently slip through. Budget 2+ sweep cycles minimum.

**§13 Pre-Delivery Sweep — add: Manual eyes-on AND automated sweep both required.** They catch different bug classes. Manual surfaces UX-felt + credibility-sensitive content issues. Automated surfaces structural + edge-case issues. Skipping either leaves a class of bugs in the ship.

**NEW §13.5 Wait-One-Natural-Cadence Run Before Client Share.** After heavy-deploy day, never share URL even if "ready" — live state reflects dev churn (quota burns, off-cadence timestamps, manual-trigger artifacts). 12-24 hour delay always worth it.

### Group C — Voice rules in §14 (3)

- **Default to 75-100 word delivery emails for time-constrained stakeholders.** Cut anything they can discover from the artifact itself. Keep: URL, access mechanism, refresh cadence, known scope gaps, invitation for input, reply path. Drop: layout descriptions, feature lists, motivational copy, "we're excited" register.
- **Establish from-address before drafting outbound copy.** `hello@` (team mailbox) vs personal mailbox determines voice register, plural vs singular, implied response team. Translating draft to different voice is brittle.
- **Pre-empt known scope gaps in delivery comms; don't sandbag.** Disclosing gaps proactively earns credibility credit. Sandbagging triggers a credibility trap when stakeholder finds them unprompted.

### Group D — /cook scaffolding additions (3)

- **Defensive normalizer layer.** For governance/agenda/multi-source projects, /cook Phase 4 scaffolds `src/normalizers.py` with no-op stubs per scraper source + regex validation on date/URL/identifier fields. Pre-built home for upstream-quirk fixes.
- **`content_hash` (or stable ID) populated at ingestion-write boundary.** For pipelines persisting items, every item carries stable identifier from moment-of-entry. Add unit test asserting non-null. Treat absent as contract violation.
- **Handoff template bundle (9 sections).** Resume Command paste-able paragraph; Status snapshot; What's New per-commit annotated; **Design Rationale** [REQUIRED at major pivots]; **Pre-flight Gaps** [REQUIRED at major pivots]; Plan with phase-numbered steps + skill/agent-by-step table; Skipped tools with reason; Gotchas for Next Session (incl. parallel-session annotation when detected); Verification commands paste-able. Add to `client-engagement-CLAUDE-template.md` Required Rules.

### Group E — Magnum Opus universal patterns (2)

- **Expose validation predicate's mechanism (not just rule) to the LLM.** When LLM output is post-validated by code, the system prompt should describe how validation runs (e.g., `display_text in raw_text` whitespace-normalized substring check) + ❌/✅ examples mimicking field-observed failure modes. Models pattern-match against shapes more reliably than abstract directives.
- **Quantitative pass gate beats self-grading evaluator-optimizer loops.** Define pass criteria with weights + numeric pass gate before dispatching evaluation. Score makes deltas legible across iterations; weights make it obvious which fixes matter. Force structured deductions tied to specific evidence.
